La Historia De HTML


HTML (HyperText Markup Language) es el lenguaje estándar utilizado para crear y estructurar contenido en la web. Su historia comienza en 1989 cuando Tim Berners-Lee, un científico británico, trabajaba en el CERN (Organización Europea para la Investigación Nuclear). Berners-Lee propuso un sistema para compartir documentos científicos utilizando hipervínculos, lo que llevó a la creación de la World Wide Web (WWW).
En 1991, Berners-Lee publicó la primera versión de HTML, conocida como HTML 1.0. Esta versión era bastante simple, con un conjunto limitado de elementos, pero sentó las bases para el desarrollo de la web. HTML 2.0, publicado en 1995, fue la primera especificación formalizada por el grupo de trabajo de la Internet Engineering Task Force (IETF). Añadió características adicionales y mejoró la estructura del lenguaje.
HTML 3.2 fue lanzado en 1997 por el World Wide Web Consortium (W3C), una organización que Berners-Lee fundó para estandarizar las tecnologías web. HTML 4.0, introducido en 1997, fue un paso importante al introducir soporte para scripts, hojas de estilo (CSS), y mejorar la accesibilidad web.
A principios de los 2000, XHTML fue desarrollado como una extensión de HTML que incorporaba reglas más estrictas basadas en XML. Sin embargo, la complejidad de XHTML y la falta de adopción generalizada llevaron al desarrollo de HTML5, que se convirtió en la nueva norma.
HTML5, lanzado oficialmente en 2014, introdujo nuevas características significativas como soporte para multimedia (video y audio) sin necesidad de plugins adicionales, elementos semánticos mejorados, y un mayor enfoque en la interoperabilidad y la accesibilidad. Esta versión marcó un punto de inflexión en el desarrollo web, permitiendo aplicaciones web más ricas y dinámicas.
Hoy en día, HTML sigue evolucionando bajo la supervisión del W3C y WHATWG (Web Hypertext Application Technology Working Group), con un enfoque continuo en mejorar la interoperabilidad, la accesibilidad y la compatibilidad con las nuevas tecnologías emergentes en la web.

Etiquetas de orden

Son etiquetas que no se ven visualmente reflejadas siempre, aqui encontramos etiquetas como footer, hgroup, div, article y otras mas, la verdad yo no las utilizo (solo el div) pero pues dicen por ahi que un buen programador las utiliza. No se total, ya mejor voy a ser licenciado

header: la etiqueta de encabezamiento define el contenido introductorio de una página o sección.
nav: la etiqueta de navegación se utiliza para los enlaces de navegación. Puede anidarse dentro de la etiqueta header, pero las etiquetas nav de navegación secundaria también suelen usarse en otras secciones de la página.
main: esta etiqueta contiene el contenido principal (también llamado cuerpo) de una página. Solo debería haber una etiqueta main por página.
article: la etiqueta artículo define el contenido independiente de la página o web en la que se encuentra. No significa necesariamente una "entrada del blog". Piensa en ella más bien como "una prenda de vestir": una etiqueta independiente que puede emplearse en varios contextos.
section: utilizar section es una forma de agrupar contenidos que tratan sobre temas similares. Las etiquetas de sección son diferentes a las de artículo, no son necesariamente autónomas, sino que forman parte de algo más.
aside: un elemento "aside" define el contenido de menor importancia. Suele emplearse para las barras laterales, que incluyen información complementaria pero no esencial.
footer: usa footer en la parte inferior de una página. La etiqueta suele incluir información de contacto, derechos de autor y navegación por la web.
